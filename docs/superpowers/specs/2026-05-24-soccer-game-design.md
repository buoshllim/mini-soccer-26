# Soccer Game Design Spec
**Date:** 2026-05-24  
**Project:** 5v5 Real-time Multiplayer Soccer Game  
**Stack:** TypeScript · Three.js · Partykit · Vercel

---

## 1. 개요

아빠와 아들(만 10세)이 함께 만드는 온라인 멀티플레이어 축구 게임.  
각자 다른 디바이스(PC/모바일)에서 접속해 1v1로 대결. 각 팀 5명(골키퍼 포함), 나머지 4명은 AI.

---

## 2. 게임 규칙

- 팀 구성: 각 팀 5명 — 골키퍼 1 + 아웃필더 4 (포메이션 선택으로 역할 결정)
- 경기 시간: 전반 5분 / 후반 5분 (기본값, 방 생성 시 1~10분 선택 가능)
- 승패: 더 많이 넣은 팀 승리. 동점은 무승부.
- 조종 전환 (공격 시): 공에 가장 가까운 내 팀 선수(골키퍼 제외) 자동 조종
- 조종 전환 (수비 시): Tab키(PC) / Switch버튼(모바일)으로 수동 전환 가능
- 골키퍼: 항상 AI 제어. 단, GK가 공을 잡아 드리블 상태가 되면 플레이어가 조종 전환

## 2-1. 필드 크기 (게임 유닛 기준)

- **필드**: 가로 100 × 세로 60 유닛
- **골대**: 폭 8 유닛, 각 골라인 중앙 배치 (x=0, x=100)
- **페널티 에어리어**: 골대 중심 기준 좌우 18 유닛, 깊이 16 유닛
- **중앙선**: x=50
- **선수 반경**: 1 유닛 (충돌 기준)
- **공 반경**: 0.5 유닛

## 3. 화면 흐름 (로비 ~ 게임)

```
홈 화면
  ├── [방 만들기] → 방 코드 생성 + 대기 화면 (링크 공유)
  └── [방 참가] → 코드 입력 → 대기 화면
         ↓ (상대 접속 시)
  로비 화면 (두 명 모두 접속)
    1. 팀 색상 선택 (파랑 / 빨강 / 초록 / 노랑) — 같은 색 선택 불가
    2. 등번호 선택 (랜덤 배정 → 직접 수정 가능)
    3. 포메이션 선택 (3×3 그리드에서 4개 선택)
    4. [Ready] 버튼
         ↓ (둘 다 Ready)
  카운트다운 (3, 2, 1)
         ↓
  킥오프 → 게임
         ↓ (종료)
  결과 화면 → [같은 설정으로 재경기] / [로비로 돌아가기]
```

### 홈 화면
- 게임 제목 + [방 만들기] / [방 참가] 버튼
- [방 참가]: 6자리 코드 입력 필드

### 대기 화면
- 방 코드 표시 + 복사 버튼
- "상대방을 기다리는 중..." 스피너
- 상대 접속 시 자동으로 로비 화면 전환

### 결과 화면
- 최종 스코어 표시
- **[재경기]**: 색상·등번호·포메이션 동일하게 즉시 재시작 (카운트다운 → 킥오프)
- **[로비로 돌아가기]**: 로비 화면으로 이동, 모든 설정 초기화 후 재설정

### 로비 화면
- **팀 색상 선택**: 4가지 색상 버튼 (파랑 🔵 / 빨강 🔴 / 초록 🟢 / 노랑 🟡)
  - 상대가 선택한 색은 비활성화
- **등번호 선택**: 입장 시 1~99 중 랜덤 숫자 자동 배정
  - 숫자 클릭 → 숫자 입력 필드로 전환, 직접 수정 가능 (1~99 범위 강제)
  - 변경 없이 Ready 눌러도 무방
- **포메이션 선택** (섹션 5.15 참고)
- **[Ready]** 버튼: 색상 + 포메이션 선택 완료 후 활성화
- 상대 Ready 여부 실시간 표시
- 둘 다 Ready → 3초 카운트다운 후 킥오프

## 4. 컨트롤

### PC (키보드)

| 키 | 공 있을 때 | 공 없을 때 |
|----|-----------|-----------|
| 방향키 | 이동 | 이동 |
| Shift | 스프린트 (스태미나 소모) | 스프린트 |
| Space | 슛 (꾹 누를수록 강함) | — (비활성) |
| C | 낮은 패스 (꾹=길게) | 서서 뺏기 |
| X | 띄우는 패스 (꾹=길게) | 슬라이딩 태클 |
| Z | 스루패스 (꾹=길게) | GK 돌진 |
| Tab | 다음 선수 전환 | 다음 선수 전환 |
| Space + X (GK 근처) | 칩샷 | — |

### 모바일 버튼 (오른쪽 영역)

| 버튼 | 공 있을 때 | 공 없을 때 |
|------|-----------|-----------|
| Shoot | 슛 | 비활성 |
| C | 숏패스 | 뺏기 |
| X | 롱패스 | 태클 |
| Z | 스루패스 | GK돌진 |
| Sprint | 스프린트 | 스프린트 |
| Switch | 선수 전환 | 선수 전환 |

버튼 라벨은 공 보유 여부에 따라 실시간 변경됨.

## 5. 기능 명세

### 5.1 드리블
- 내 조종 선수가 공으로부터 1.5 게임 유닛 이내 진입 시 공이 선수 앞에 자석처럼 부착
- 이동 방향으로 공이 함께 이동
- **드리블 공 애니메이션**: 공이 앞에 붙어있는 동안 좌우로 살짝 흔들리며 바운스 (달리는 리듬에 맞춰)

### 5.2 슛
- Space / Shoot 버튼을 누르는 시간에 비례해 파워 결정 (최대 1.5초)
- 파워 게이지 UI 표시 (선수 위에 바 형태)
- 공이 붙어있는 상태에서만 발동
- **슛 방향**: 버튼 뗄 때 방향키 입력 중이면 그 방향, 방향키 없으면 선수가 바라보는 방향 (FC 방식)

### 5.3 패스
- **낮은 패스 (C)**: 지면 패스. 꾹 누를수록 거리 증가. 방향키 방향으로 전송.
- **띄우는 패스 (X)**: 공중 패스. 꾹 누를수록 높이·거리 증가. 공에 z축 속도 추가.
- **스루패스 (Z)**: 수비 뒤 공간으로 보내는 패스. 방향키 방향 기준.
- 모두 최대 충전 시간 1.5초, 파워 게이지 표시.
- **조종 전환 타이밍**: 패스 키를 뗀 순간(킥 시점)에 즉시 수신 선수로 전환 (FC 방식)

### 5.4 스프린트 & 스태미나
- Shift / Sprint 버튼 누르는 동안 이동 속도 1.6배
- 스태미나 바 표시 (선수 발 아래, 흰색 바)
- 스태미나 소진 시 스프린트 불가, 기본 속도 0.8배로 감소
- 스프린트 안 할 때 초당 20% 회복

### 5.5 수비 & 선수 전환

**선수 전환 (수동)**
- Tab키(PC) / Switch버튼(모바일): 내 팀 아웃필더 중 공에 두 번째로 가까운 선수로 전환
- 골키퍼는 수동 전환 대상에서 제외 (항상 AI)
- 연속으로 누르면 순환

**서서 뺏기 (C)**
- 내 선수가 공으로부터 2 유닛 이내일 때 발동 가능
- 발동 즉시 볼 탈취 (타이밍 판정 없음, 거리 조건만)

**슬라이딩 태클 (X)**
- 방향키 방향으로 슬라이딩
- 공에 닿으면 성공 → 볼 탈취
- 공에 닿지 못하면 실패 → 아무 일 없음
- 선수 뒤에서 발동 시 → 파울 (뒤에서 여부는 서버가 각도로 판정)

### 5.6 GK 돌진 (Rush)
- **발동**: 수비 상황(공 없을 때) Z키(PC) / Z버튼(모바일) 입력
- GK가 AI 제어로 공 방향으로 전력 질주 (FC 방식)
- **공격수가 드리블 중일 때**: GK가 도착하면 서서 뺏기 시도
  - 성공 → 공이 자유 상태 → 아래 자유 공 처리로 이어짐
  - 실패 → 공격수 돌파, GK 아웃포지션
- **공이 자유 상태일 때**: 바로 처리
  - 페널티 에어리어 안: 공을 손으로 잡음 → GK 드리블 상태 → 플레이어 조종 전환
  - 페널티 에어리어 밖: 발로 클리어 (핸드볼 판정 없음)
- 리스크: 돌진 실패 시 골문 비워짐

### 5.7 골키퍼 컨트롤
- 평상시: AI가 골문 앞 좌우 이동 + 공 방향 추적 + 돌진
- **GK 컨트롤 전환 조건**: 아래 두 조건 모두 충족 시 플레이어에게 조종 전달
  1. 공이 **자유 상태** (어느 선수에게도 붙어있지 않음)
  2. 공이 GK로부터 **1.5 유닛 이내**
  - → 세이브 후, 백패스 수신 후 모두 동일하게 적용 (FC 방식)
  - → 상대가 GK 근처까지 드리블해도 공이 상대에게 붙어있으면 전환 안 됨
- **플레이어 GK 조종 중**: C(낮은 패스) / X(띄우는 패스) / Z(스루패스/롱킥) / 방향키 이동 / Space(강킥) 가능
  - 이 상태에서 Z는 GK 돌진이 아닌 스루패스로 동작 (공을 이미 가진 상태이므로 돌진 불필요)
- **AI 복귀**: 플레이어가 공을 차는 순간 GK 다시 AI로 복귀

### 5.8 헤더
- 공 z축 높이가 2 유닛 이상이고 선수로부터 2 유닛 이내 하강 시 자동 발동
- 방향키 방향으로 공 전송 (헤딩 슛 또는 헤딩 패스)
- **GK 예외**: 공중볼이 GK 근처에 하강할 때 — 페널티 에어리어 안이면 헤더 대신 손으로 잡기 (GK 드리블 상태 전환), 에어리어 밖이면 일반 헤더

### 5.9 세트피스

| 상황 | 처리 |
|------|------|
| 파울 발생 위치 | 프리킥 (파울 지점) |
| 페널티 에어리어 내 파울 | 페널티킥 |
| 공격팀이 골라인 밖으로 냄 | 골킥 |
| 수비팀이 골라인 밖으로 냄 | 코너킥 |
| 사이드라인 밖 | 스로인 |

- **자동 배치 (FC 방식)**: 세트피스 발생 시 해당 팀 조종 선수가 공 위치 근처(1.5 유닛 뒤)에 자동 배치됨. 별도 이동 없이 바로 슛/패스 입력으로 재개
- 상대팀은 9 유닛 거리 유지 강제 (페널티킥 제외)
- 세트피스 중 화살표 UI로 재개 방향 표시

### 5.10 킥오프

- 전반 시작: 랜덤으로 한 팀이 중앙 킥오프
- 전반 종료 → 하프타임 → 반대 팀 킥오프
- 골 실점 후: 실점한 팀이 중앙에서 킥오프
- 킥오프 방식: 공이 중앙에 고정 → 조종 선수가 공 근처에서 패스 키 입력 시 게임 재개 (패스 전까지 상대팀 중앙선 침범 불가)

### 5.11 오프사이드
- 패스 시점에 상대 최후방 필드 플레이어 뒤에 위치한 공격수에게 패스 → 오프사이드
- 판정 시 간접 프리킥

### 5.12 파울 판정
- 슬라이딩 태클을 선수 뒤에서 발동 시 파울
- 파울 위치 기준 프리킥 또는 페널티킥

### 5.13 칩샷
- **PC 발동**: 상대 GK로부터 10 유닛 이내에서 Space 홀드 중 X 추가 입력
- **모바일 발동**: Shoot 버튼 꾹 누르는 동안 X 버튼 라벨이 "칩샷"으로 변경 → X 버튼 탭
- 공이 낮은 포물선으로 GK 위를 넘어가는 슛
- 파워는 홀드 시간과 무관하게 고정 (너무 강하면 안 넘어감)

### 5.14 볼 쉴딩
- **발동 조건**: 드리블 중 + 상대 선수가 2 유닛 이내 + 방향키를 상대 반대 방향으로 입력 (뒤로 버팀)
- 서버가 상대 위치 기준으로 쉴딩 여부 판정
- 쉴딩 상태에서 상대의 서서 뺏기 성공률 50% 감소
- 쉴딩 중 이동 속도 0.6배로 감소

### 5.15 포메이션 선택

**UI: 3×3 그리드 (9개 원)**

```
[ 1 ] [ 2 ] [ 3 ]   ← Row 1: 공격수 (FWD)
[ 4 ] [ 5 ] [ 6 ]   ← Row 2: 미드필더 (MID)
[ 7 ] [ 8 ] [ 9 ]   ← Row 3: 수비수 (DEF)
```

- 정확히 4개 선택해야 Ready 버튼 활성화
- 선택한 원의 행(Row)이 해당 선수의 역할을 결정
- 원의 열(Column)이 필드에서의 좌우 초기 위치를 결정
- 골키퍼는 그리드 외부에 고정 표시 (선택 불가)

**포메이션 예시**

| 선택 | 구성 | 형태 |
|------|------|------|
| 1,3,7,9 | FWD 2 + DEF 2 | 2-0-2 |
| 1,3,5,7 | FWD 2 + MID 1 + DEF 1 | 2-1-1 |
| 2,4,6,8 | FWD 1 + MID 2 + DEF 1 | 1-2-1 |
| 4,5,6,8 | MID 3 + DEF 1 | 0-3-1 |

**미드필더 AI 행동**
- 공이 상대 진영에 있을 때 → 공격수처럼 전진
- 공이 우리 진영에 있을 때 → 수비수처럼 후퇴
- 기준선: 필드 중앙선

## 6. AI

### 역할별 AI 행동

| 역할 | 기본 포지션 | 공격 시 | 수비 시 |
|------|-----------|---------|---------|
| FWD | 상대 진영 | 공 쫓아가 슛 시도 (15유닛 이내) | 상대 진영 유지 |
| MID | 중앙선 부근 | 공격 가담 (전진) | 수비 가담 (후퇴) |
| DEF | 우리 진영 | 우리 진영 유지 | 공과 골문 사이 포지셔닝, 5유닛 이내 서서 뺏기 |

### 골키퍼 AI
- 골문 앞 좌우 이동 (공 방향 추적, 골문 라인 이탈 안함)
- 공이 골문으로부터 8 유닛 이내 진입 시 돌진해서 차냄
- 세이브 후 공이 1.5 유닛 이내 → 플레이어에게 컨트롤 전달

## 7. 그래픽 & 카메라

- **렌더러**: Three.js WebGL
- **카메라**: Perspective, 와이드 앵글 (FOV 65°)
- **동적 줌**: 공 위치에 따라 카메라가 공 쪽으로 부드럽게 이동·줌인 (lerp)
- **필드**: 3D 평면, 잔디 텍스처, 라인 마킹, 3D 골대

### 캐릭터 (로우폴리)
- 머리(구) + 몸통(박스) + 팔 2개 + 다리 2개
- 눈·입 텍스처 (코 없음)
- **애니메이션**: 달리기/서기/킥/태클 시 팔다리 프로그래매틱 스윙 (사인파 기반)
- 팀 구분: 유니폼 색상 (파랑 / 빨강 / 초록 / 노랑)
- **등번호 표시** (조종 선수 + AI 전원):
  - 몸통(등 쪽 면): 흰색 텍스처에 등번호 렌더링 (Canvas → Three.js 텍스처)
  - 머리 위: 빌보드 스프라이트로 등번호 숫자 표시 (항상 카메라를 향함)
- 조종 중인 선수 표시: 머리 위 작은 화살표 ▼ (등번호 위에 추가 표시)

### 공
- 3D 구, 흑백 패널 텍스처
- 드리블 중: 좌우 흔들림 + 바운스 애니메이션
- 공중 패스 시: z축 포물선 이동, 지면 그림자로 높이 표현

## 9. 네트워크

- **서버**: Partykit (TypeScript, Durable Objects)
- **게임 루프**: 서버에서 20틱/초 실행
- **서버 권위적**: 물리 연산, 충돌, 골 판정, 파울 판정, 오프사이드 판정 모두 서버
- **클라이언트**: 입력만 전송, 서버 상태 받아서 렌더링 (클라이언트 예측 없음)
- **방 시스템**: 6자리 랜덤 코드로 방 생성, URL `?room=ABC123` 형식

### 공유 타입

```typescript
type Vec2 = { x: number; y: number }
type Vec3 = { x: number; y: number; z: number }
type TeamColor = 'blue' | 'red' | 'green' | 'yellow'
type PlayerRole = 'gk' | 'fwd' | 'mid' | 'def'

type Player = {
  id: string
  team: 'home' | 'away'
  role: PlayerRole
  pos: Vec2
  stamina: number       // 0~1
  isControlled: boolean
  hasBall: boolean
  jerseyNumber: number  // 1~99 (조종 선수: 로비에서 설정, AI: 서버가 랜덤 배정, 중복 허용)
}

type Ball = {
  pos: Vec3
  vel: Vec3
}

type Formation = {
  slots: number[]       // 선택된 그리드 인덱스 (0~8), 정확히 4개
}

type LobbyState = {
  home: { color: TeamColor | null; formation: Formation | null; jerseyNumber: number; ready: boolean } | null
  away: { color: TeamColor | null; formation: Formation | null; jerseyNumber: number; ready: boolean } | null
}

type SetpieceState = {
  type: 'freekick' | 'penalty' | 'corner' | 'throwin' | 'goalkick'
  team: 'home' | 'away'
  pos: Vec2
}

type GameStats = {
  possession: { home: number; away: number }  // 누적 틱 수 (공 소유 팀 기준)
  shots: { home: number; away: number }        // 슈팅 횟수
  shotsOnTarget: { home: number; away: number } // 유효 슈팅 횟수
}

type GameState = {
  players: Player[]
  ball: Ball
  score: { home: number; away: number }
  timeLeft: number
  half: 1 | 2
  phase: 'lobby' | 'kickoff' | 'playing' | 'freekick' | 'penalty' | 'corner' | 'throwin' | 'goalkick' | 'halftime' | 'ended'
  kickoffTeam: 'home' | 'away' | null
  setpiece?: SetpieceState
  lobby?: LobbyState
  stats: GameStats
}

type PlayerInput = {
  dx: number              // -1, 0, 1
  dy: number              // -1, 0, 1
  sprint: boolean
  switchPlayer: boolean   // Tab 키
  action: 'shoot' | 'chipshot' | 'lowpass' | 'loftedpass' | 'throughpass' | 'tackle' | 'slidetackle' | 'gkrush' | null
  power: number           // 0~1, 버튼 누른 시간 비율
}
```
